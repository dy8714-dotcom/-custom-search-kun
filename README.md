# カスタマイズ検索くん

複数の検索サイトを効率的に使用できるカスタマイズ可能な検索ツールです。

## 🎯 機能概要

### 現在実装済みの機能

#### ✅ メイン機能
- **10個の検索窓**: 複数のキーワードを同時に管理・検索
- **一括入力機能（NEW!）**: カンマ・スペース・読点区切りで最大10個を一度に入力
- **リセット機能（NEW!）**: すべての検索窓を一括クリア
- **カスタマイズ可能な検索サイト**: 最大10個まで任意の検索サイトを登録
- **複数サイト同時検索**: チェックボックスで選択した複数サイトで一度に検索
- **一括検索**: 10個の検索窓すべてを一度に検索（1サイトのみ選択時）
- **Google検索**: シンプルなGoogle検索機能
- **キーボードナビゲーション**: Tab/Shift+Tabで検索窓を移動

#### ✅ データ管理
- **インポート/エクスポート**: JSON形式でカスタマイズ設定を保存・読込
- **ローカルストレージ**: ブラウザに設定を自動保存
- **サイト編集**: モーダルウィンドウで直感的に編集

## 📖 使い方

### 基本的な検索

1. **検索窓に検索キーワードを入力**
2. **右側のサイトリストから検索したいサイトにチェック**
3. **検索ボタンをクリック**または**Enterキー**を押す
4. チェックしたサイトで別タブが開きます

### 一括入力機能（NEW!）

複数のキーワードを一度に10個の検索窓に入力できます。

1. **ヘッダーの「一括検索入力」窓に入力**
2. キーワードを区切り文字で区切って入力：
   - `,` (半角カンマ)
   - `，` (全角カンマ)
   - `、` (読点)
   - ` ` (スペース)
3. **「入力」ボタン**をクリック（またはEnter）
4. 自動的に上から順番に検索窓に入力されます

**例:**
```
入力: 7203,6758,9984
または: 7203 6758 9984
または: 7203、6758、9984
↓
検索窓1: 7203
検索窓2: 6758
検索窓3: 9984
```

💡 **ポイント:**
- 最大10個まで入力可能
- 既に検索窓に入力されていても上書きされます
- 前後のスペースは自動削除されます

### リセット機能（NEW!）

**「リセット」ボタン**をクリックすると、すべての検索窓（10個+Google検索窓+一括入力窓）が一度にクリアされます。

### 複数サイト同時検索

- 複数のサイトにチェックを入れて検索すると、すべてのサイトで同時に検索されます
- 例: 3つのサイトにチェック → 3つのタブが開きます

### 一括検索

1. **10個の検索窓にキーワードを入力**
2. **検索サイトを1つだけ選択**（重要！）
3. **一括検索ボタン**をクリック
4. 確認ダイアログで「OK」をクリック
5. 入力されたすべてのキーワードで順次検索されます

⚠️ **ポップアップブロックの解除方法**: 

> 📘 **詳しい解除手順は [POPUP-HELP.md](POPUP-HELP.md) をご覧ください**

**初回使用時、タブが開かない場合:**

1. ブラウザのアドレスバーの右側に「ポップアップがブロックされました」というアイコンが表示されます
2. そのアイコンをクリック
3. 「常に許可」または「ポップアップとリダイレクトを許可」を選択
4. もう一度「一括検索」ボタンをクリック

**ブラウザ別の設定:**

- **Chrome/Edge**: アドレスバー右の🚫アイコン → 「常に許可」
- **Firefox**: アドレスバー左の盾アイコン → 設定を変更
- **Safari**: Safari環境設定 → Webサイト → ポップアップウインドウ → 「許可」

💡 **一度許可すれば、次回からは自動的にすべてのタブが開きます**

⚠️ **注意**: 
- 一括検索は1つのサイトのみ選択時のみ動作します（タブの大量オープンを防止）
- 複数のタブが開くため、PCの負荷にご注意ください

### サイトのカスタマイズ

1. **ページ追加・編集ボタン**をクリック
2. モーダルウィンドウが開きます
3. サイト名とURLを編集（URLに`TESTSEARCH`を含める）
4. **サイトを追加ボタン**で新しいサイトを追加（最大10個）
5. **保存ボタン**をクリック

#### URL形式の例
```
https://example.com/search?q=TESTSEARCH
https://example.com/find/TESTSEARCH
https://example.com?keyword=TESTSEARCH
```

`TESTSEARCH`の部分が実際の検索キーワードに置き換わります。

### データのバックアップ

#### エクスポート
1. **エクスポートボタン**をクリック
2. `custom-search-sites.json`ファイルがダウンロードされます

#### インポート
1. **インポートボタン**をクリック
2. JSONファイルを選択
3. サイト設定が復元されます

### キーボード操作

- **Tab**: 次の検索窓に移動
- **Shift+Tab**: 前の検索窓に移動
- **Enter**: 現在の検索窓で検索を実行

## 🚀 デプロイ（GitHub Pagesで公開）

> 📘 **詳細な手順は [DEPLOY.md](DEPLOY.md) をご覧ください**

### GitHub Pagesで公開する手順（概要）

1. **GitHubにリポジトリを作成**
   - GitHubにログインし、新しいリポジトリを作成します
   - リポジトリ名を入力（例：`custom-search-kun`）
   - Public（公開）を選択

2. **コードをプッシュ**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/あなたのユーザー名/リポジトリ名.git
   git push -u origin main
   ```

3. **GitHub Pagesを有効化**
   - リポジトリの「Settings」タブをクリック
   - 左メニューから「Pages」を選択
   - Source（ソース）で「main」ブランチを選択
   - Saveをクリック
   - 数分後、公開URLが表示されます

4. **公開URLにアクセス**
   - `https://あなたのユーザー名.github.io/リポジトリ名/`でアクセス可能になります

### その他のデプロイ先

静的サイトのため、以下のサービスでも簡単に公開できます：

- **Netlify**: ドラッグ&ドロップで即座にデプロイ
- **Vercel**: GitHubと連携して自動デプロイ
- **Firebase Hosting**: Googleのホスティングサービス
- **Cloudflare Pages**: 高速なCDN配信
